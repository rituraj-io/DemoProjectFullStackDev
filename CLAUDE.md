# Coding Instructions

- Follow the **DRY** principle — no repeated logic; extract reusable code.

- Write **clean, readable code** with clear structure.

- Add **user-friendly comments** that explain intent, not just what the code does.

- Use **JSDoc** for all documentation (types, params, returns).

- Separate logical sections with **double newlines**.

- Use **4 spaces** (or 1 tab) for indentation.

- Follow **Prettier** formatting rules.

- Always run `tsc` before confirming a work as DONE and fix type errors

- Never include **Co-Authored-By** or self-attribution in git commits.
